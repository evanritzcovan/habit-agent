🟢 PHASE 6 — AI Edge Function

Task 20
Create Supabase Edge Function for plan generation:
- Input: habit_name, habit_type, user_input (e.g. optional context at create), habit_id (for attribution)
- For initial generation: no “difficulty” from the user; prompt the model to pick a reasonable default and emit `difficulty` inside the plan JSON only
- For regenerate / adjust: include optional `adjustment` { level: "too_easy" | "just_right" | "too_hard", optional free_text } and (for pure regenerate) any extra user string; use these to steer intensity
- Secrets: OpenAI API key in Supabase secrets; use a defined model (e.g. gpt-4o-mini — confirm in function comments / env)
- Process: call OpenAI; parse with shared Zod schema; retry once on invalid output
- Enforce global generation allowance for free tier (3 per calendar month for non-subscribers, per config/product.json): read/increment ai_generation_usage within the same transaction or secure pattern; return error when over limit
- Return validated JSON plan

Task 21 — Phase checkpoint
Invoke function from Supabase client with test user; invalid JSON path retries; limit exceeded returns clear error.
